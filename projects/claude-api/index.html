Getting Started

Include the Puter runtime, then call Claude from the browser. The page below already loads the runtime and exposes a live console so you can verify the flow without scaffolding a separate app.

HTML bootstrap 
<script src="https://js.puter.com/v2/"></script>

Example 1: Basic Text Generation with Claude Sonnet 4.6

Minimal call path: prompt plus explicit model selection. The playground uses the same API surface and normalizes the returned content into the transcript pane.

Basic completion 
puter.ai.chat("Explain quantum computing in simple terms", { model: "claude-sonnet-4-6" })
  .then((response) => {
    puter.print(response.message.content[0].text);
  });

Example 2: Streaming Responses for Longer Queries

Streaming is exposed through stream: true. When enabled in the lab, the assistant bubble is incrementally appended as chunks arrive.

Streaming completion 
const response = await puter.ai.chat(
  "Write a detailed essay on the impact of artificial intelligence on society",
  { model: "claude-sonnet-4-6", stream: true }
);

for await (const part of response) {
  puter.print(part?.text);
}

Example 3: Switching Claude Models

  • Haiku path: set model: "claude-haiku-4-5" for shorter, faster responses.
  • Opus path: set model: "claude-opus-4-6" when you want higher-capacity reasoning in the same frontend flow.
  • Sonnet path: default the lab to claude-sonnet-4-6 to match the tutorial’s mainline example.

How to use this page

  1. Pick a Claude model and whether to stream.
  2. Run a preset or paste your own prompt.
  3. If Puter requires auth in-browser, complete that flow and retry.
  4. Copy the generated code sample if you want to move the request into your own HTML file.

Operational caveat

This page reproduces the user-facing integration pattern from the provided tutorial. Actual availability, auth prompts, quotas, and runtime behavior are controlled by Puter’s service, not by this static site.
