Bootstrap

Runtime setup is still one script tag. The material change is request composition in the client, not backend infrastructure.

HTML bootstrap 
<script src="https://js.puter.com/v2/"></script>

What changed

  • Prompt composer: request body is assembled from system instructions, ephemeral context, selected markdown packs, attachment excerpts, and the operator’s prompt.
  • Attachment normalization: readable files are parsed in-browser and clipped into deterministic excerpts; unreadable files contribute metadata only.
  • Markdown context library: custom .md files can be imported, activated, persisted, exported, and reused across sessions.

Streaming call path

The execution primitive is unchanged: puter.ai.chat(). The difference is that the request string now carries richer structured context assembled on the page.

Streaming completion 
const response = await puter.ai.chat(composedPrompt, {
  model: "claude-sonnet-4-6",
  stream: true,
});

for await (const part of response) {
  puter.print(part?.text);
}

How to use this page

  1. Pick a Claude model and optional streaming.
  2. Load reusable markdown context packs or import new ones from disk.
  3. Drop files into the attachment lane if the request should cite local material.
  4. Review the generated snippet and composed size, then run the request.

Operational caveat

This site is still constrained by Puter’s runtime semantics, auth flow, quotas, and model availability. File uploads and context packs are composed into a prompt string in-browser; they are not transmitted as raw multipart uploads to a Claude-native API.
